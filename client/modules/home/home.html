About this site

If you landed here looking for someone named Brandon Lind and he isn’t a software architect / developer, you’re probably in the wrong place ;).

I use this site as a sandbox to keep current with emerging technologies and theories in web architecture and software development. As such, it’s usually pretty plain-vanilla, but under the covers, I’m practicing everything from developing using a cloud IDE (such as Cloud9 or Koding) to using Github and deploying to Heroku.

The server software also changes constantly. Sometime this site is developed using Ruby on Rails, sometimes Nodejs. Typically for a persistence engine, I stick with MongoDB.

The JavaScript framework also changes constantly, from Backbone to Knockout, Angular to KendoUI, it’s always in flux and a majority of my changes are to this area.
